WatchTower

Observation-First Agent Security — Taint Propagation and Semantic Enforcement in Multi-Agent Systems

Rohit Jinsiwale Reliance Industries June 2026
IEEE S&P USENIX Security ACM CCS

Abstract

Multi-agent LLM systems expose three distinct attack surfaces that existing defenses address in isolation, leaving the threat landscape largely uncovered. Surface 1 — Input Corruption: MINJA achieves a 98.2% memory-injection rate via query-only interaction, with no write access required; standard content filters carry no memory across sessions. Surface 2 — Capability Abuse: legitimate tool calls (e.g., send_email) become attack vectors when invoked after sensitive reads — indistinguishable at the wire layer from benign use. Surface 3 — Multi-Agent Contagion: a compromised agent writes poisoned data to shared memory; downstream agents inherit taint through read-only queries. No prior defense covers this surface.

WatchTower is the first system to address all three surfaces simultaneously. It interposes a 9-layer hot-path pipeline — deterministic rule evaluation with AST call-tree context, taint-propagation scoring, and cryptographic identity — achieving 17/17 detection on a known-bad corpus at 0.011 ms p99 latency with zero false positives, running 9,636× faster than state-of-the-art GPU-based alternatives.

Architecture

WatchTower interposes on every agent tool call via a 9-layer pipeline. Layers L0–L5 execute synchronously on the hot path; L6–L7 run asynchronously behind an asyncio barrier. Any error at any layer defaults to BLOCK (fail-safe). The pipeline completes a full evaluation — identity check, AST enrichment, deterministic rules, and trust scoring — in under 0.011 ms p99.

Two-Tier Enforcement

The hot tier (L0–L5) delivers a synchronous verdict in microseconds using deterministic YAML rules evaluated against an AST call tree. For ambiguous cases, L5 defers to the cold tier (L6–L7): a Byzantine-fault-tolerant swarm of three specialist agents that reach consensus within 5 seconds. Result delivery is held at the L7 barrier until the cold verdict arrives — ensuring no tool call escapes evaluation.

Why call_tree_contains Matters

Wire-level firewalls see the tool name and arguments but not why the call was made. WatchTower's L2 enrichment builds a call-tree fingerprint for every invocation; L3 rules can then predicate on semantic context — blocking send_email only when it follows read_secret in the call chain.

Taint Model

WatchTower tracks taint as a real-valued score in [0, 1] per agent, persisted in cavemem (the shared memory service). Propagation follows two equations: a hop-decay rule when agent B reads memory written by tainted agent A, and a time-decay rule that clears taint exponentially.

Taint Propagation Equations T(B) = max(T_B, T_A × ρ) // hop decay on read from tainted writer T(t) = T₀ × e−λt // time decay; expires ~2.5 hr for T₀=0.9 trust′ = trust × (1 − β × T) // trust degradation on taint event trust′ = trust + α × (1 − trust) // trust recovery per clean call

ρ0.8Hop decay factorT_B = T_A × 0.8 per read hop
λ0.1 / hrTime decay rateHalf-life ≈ 6.9 hr; clears below Q in ~2.5 hr for T₀=0.9
α0.02Recovery rateTrust += 0.02 × (1 − trust) per clean call
β0.6Degradation factortrust ×= (1 − 0.6 × T) on taint event
Q0.7Quarantine thresholdT ≥ 0.7 → agent quarantined, all tool calls blocked

Quarantine recovery: an agent exits quarantine when its taint decays below Q=0.7 via time-decay (approximately 2.5 hours for a freshly tainted agent at T₀=0.9), or when explicitly cleared by an operator after manual review.

Performance Benchmark

N=1000 concurrent evaluations of the L3 deterministic rule engine plus in-process trust lookup, measured on a single commodity core without GPU. The 0.011 ms p99 result is 1000× inside the 10 ms SLO and 9,636× faster than the nearest GPU-accelerated competitor (Distributed Sentinel, 106 ms).
