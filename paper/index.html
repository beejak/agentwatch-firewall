Abstract

The Problem No One Has Fully Solved

Multi-agent AI systems face three attack surfaces that existing defenses address only in isolation. WatchTower is the first published system to address all three — simultaneously — with cross-session taint tracking.

💉
Input Corruption
Adversary injects instructions into content the agent reads. MINJA achieves this via query-only interaction — no write access required.
98.2% injection rate
🔧
Capability Abuse
Legitimate tool, illegitimate purpose. send_email after read_secret — same API, completely different intent. Wire-level can't tell the difference.
Undetectable at network layer
🦠
Multi-Agent Contagion
Compromised agent writes to shared memory. Downstream agents inherit taint through read-only queries. No published defense existed.

The Semantic Gap — Why call_tree_contains Changes Everything

wire-level enforcement (Claw Patrol)
# Network layer sees endpoint, not intent send_email(to="partner@corp.com") # → ALLOW (looks legitimate) send_email(to="partner@corp.com", body="api_key=sk-abc123") # → ALLOW (same endpoint! blindspot) # Wire-level cannot distinguish these. # Both hit the same SMTP gateway.
exfil_email.yaml (WatchTower)
rule: block_external_email_with_secrets match: tool: send_email context: # semantic: why was this called? call_tree_contains: [read_secret] any: - arg.to: { not_in_domain: ["${ORG_DOMAIN}"] } - arg.body: { matches_secret_pattern: true } verdict: BLOCK # → BLOCK (read_secret in call tree = exfil)
