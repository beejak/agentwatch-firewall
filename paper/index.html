Abstract

The Problem No One Has Fully Solved

Three attack surfaces. Existing systems cover one each. WatchTower covers all three — with cross-session taint tracking no one else has.

💉
Input Corruption
MINJA achieves this via query-only interaction — no write access required. Standard content filters have no persistence across sessions.

Capability Abuse
Legitimate tool, illegitimate purpose. send_email after read_secret — same API endpoint, completely different intent.
Invisible at wire layer

Multi-Agent Contagion
Compromised agent writes shared memory. Downstream agents inherit taint through read-only queries. No prior defense.

Why call_tree_contains changes everything

wire-level (Claw Patrol) — blind to intent
# Network layer sees endpoint, not intent send_email(to="partner@corp.com") # → ALLOW (legitimate) send_email(to="partner@corp.com", body="api_key=sk-abc123") # → ALLOW ← BLINDSPOT: same endpoint!
exfil_email.yaml (WatchTower) — semantic
rule: block_external_email_with_secrets match: tool: send_email context: # WHY was this called? call_tree_contains: [read_secret] any: - arg.body: { matches_secret_pattern: true } verdict: BLOCK # intent matters
